# OutputLens Component Standards

## Analysis Card

Must display:

- Asset
- Direction
- VaR
- Risk Score
- Win Probability

Never remove risk metrics.

---

## Risk Metric Card

Must include:

- Metric name
- Metric value
- Plain English explanation

Optional:

- Trend
- Delta

---

## Dashboard Cards

Must answer:

"What action should the user take?"

Not:

"What looks visually impressive?"

---

## Scenario Cards

Must display:

- Probability
- Outcome range
- Regime

Never:

- Show predictions
- Show guarantees

---

## Regime Cards

Required:

- Current regime
- Confidence
- Regime description

---

## Tables

Preferred over excessive cards when data density improves usability.

Institutional users value information density.

---

## Modals

Use only when necessary.

Avoid modal-heavy workflows.

---

## Empty States

Should educate users.

Example:

"No analyses yet. Run your first simulation to generate risk insights."
